# Implementation Plan

## Research Notes

OpenRouter's official docs describe the Chat Completions endpoint at `/api/v1/chat/completions`, JSON Schema structured outputs through `response_format`, model metadata through `GET /api/v1/models`, and provider routing controls including `order`, `allow_fallbacks`, `require_parameters`, `data_collection`, and endpoint-specific provider slugs.

The 2026-06-24 source refresh also confirmed that managed OpenRouter Fusion currently includes a retrieval-enabled inner deliberation path: panel and judge calls can use `openrouter:web_search` and `openrouter:web_fetch`, with configurable panel/judge/tool-loop knobs through the beta server tool. Frugal Fusion should continue treating managed Fusion as an external reference condition unless future runs introduce matched retrieval budgets for every baseline.

See `docs/RESEARCH_NOTES.md` for the current Fugu/Fusion/self-consistency/judge-bias research summary.

## Public-Readiness Delta

The initial MVP proves the local shape but is not yet credible as a public cost-performance experiment. The next changes should stay small and target these gaps:

- Move model and budget settings out of CLI constants. Status: implemented with `src/config.ts` and `examples/frugal-fusion.config.json`.
- Add preflight cost estimates so a panel cannot obviously exceed budget before calls are issued. Status: implemented at the orchestration boundary.
- Add self-review and same-model repeated-sampling baselines to evaluation. Status: implemented as `self_review` and `repeated`.
- Fix trace privacy so raw tasks are not retained by default. Status: implemented; raw prompt retention is opt-in.
- Randomize or otherwise blind candidate order before aggregation and never expose model names or prices to the aggregator. Status: implemented with deterministic per-task candidate aliasing/order.
- Record prompt versions, configuration IDs, invalid-output rates, and failure rates in traces. Status: implemented for local traces and aggregate metrics.
- Keep adaptive routing, learned coordination, search, and a third candidate out until traces justify them.
- Make `auto` honest instead of adaptive. Status: implemented as a direct-only MVP placeholder with closed-enum `metadata.autoRouting`; no heuristic router is enabled.
- Add a public-report redaction layer so shareable artifacts do not expose private prompts, case IDs, model/provider IDs, traces, or answer keys. Status: implemented with `src/publicReport.ts` and `eval --public-out`.
- Add a larger public sample case set and no-spend case validation path. Status: implemented with `examples/cases.public.jsonl`, `validateEvalCases`, and `validate-cases`.
- Add a deterministic public sample manifest without weakening public-report privacy. Status: implemented with canonical case-set fingerprints, `validate-cases --manifest-out`, and `examples/cases.public.manifest.json`.
- Add descriptive public category breakdowns without exposing raw labels. Status: implemented with generated category IDs, all-or-nothing small-category suppression, explicit denominators, and grader/check-kind counts.
- Add machine-readable grader evidence tiers so public readers can see when results are structured/exact versus surface-text heavy without exposing raw grader values. Status: implemented as pre-spend derived metadata in validation summaries, manifests, private evaluation cases, and public category reports.
- Add a low-cost citation-mechanics grader for source-id discipline without claiming semantic entailment. Status: implemented with bracket citation ID validation, required cited sources, minimum cited-source counts, literal claim/citation proximity checks, optional immediate citation placement for required literal claims, symbolic diagnostics, and public check-kind projection.
- Add private-audit HMAC manifests for holdout identity without publishing unsalted prompt/grader digests. Status: implemented as `validate-cases --manifest-hmac-key-env`, with keyed digest fields and disclosure-label guards.
- Add a no-spend case-set claim gate so CI can block premature public cost-performance claims without implying claim approval. Status: implemented as `validate-cases --claim-gate public_cost_performance`, with public-only JSON output, exit code 2 on blockers, skipped manifest output on blockers, and `overallClaimStatus: "external_evidence_required"` even when case-set constraints are met.
- Add a public-report evidence gate so shared reports show whether fixed MVP matrix evidence, direct-vs-fusion pairing, intervals, cost evidence, category floors, and no-digest private-report attestations are present without implying claim approval. Status: implemented in public report schema `frugal-fusion-public-eval-v11` as `claimGate`, derived from public projection fields plus no-digest private case-set/run attestations and always carrying `overallClaimStatus: "external_evidence_required"`; v11 includes descriptive category-level task pass-rate and pass-rate-delta intervals, scored-attempt coverage diagnostics, public choice check kinds, no-digest OpenRouter request-policy disclosure, and no-digest provider endpoint pinning disclosure while keeping category cost intervals omitted.
- Add a grader-evidence floor to claim gates so scored evidence with fewer than five non-surface grader-evidence cases cannot pass public cost-performance constraints. Status: implemented for both no-spend case-set gates and public-report gates with aggregate-only counts, without exposing raw grader values.
- Add a category-level grader-evidence floor so non-surface evidence cannot be concentrated in one category while other claim-eligible categories remain surface-text-only. Status: implemented for both no-spend case-set gates and public-report gates with aggregate-only blocker evidence and additive `claimGate.minimums` fields under public report schema v5.
- Add exact and high-confidence lexical near-duplicate scored-content hygiene so repeated or heavily templated task/constraint content cannot inflate no-spend case-set claim gates. Status: implemented in `validate-cases --claim-gate public_cost_performance` with internal exact content fingerprints that ignore case IDs and categories, deterministic task/constraint token-set near-duplicate checks that do not use grader values, aggregate-only blocker evidence, smoke-only cases excluded, and no manifest or public-report schema changes.
- Add closed-enum difficulty coverage metadata so public cost-performance case-set gates cannot pass with unlabelled or single-difficulty holdouts. Status: implemented as optional `difficulty: "easy" | "medium" | "hard"` case metadata, v4 case-set manifests with difficulty in canonical identity, aggregate-only difficulty coverage summaries, no row-level difficulty disclosure in manifests, no execution-seed changes, and a no-spend claim-gate floor of 30 scored cases per difficulty bucket.
- Add aggregate-only category-by-difficulty coverage checks so public cost-performance case-set gates cannot pass when difficulty metadata is concentrated in only a few claim-eligible categories. Status: implemented in the no-spend case-set gate with a mechanical floor of five scored cases per closed-enum difficulty bucket within each claim-eligible category, aggregate-only blocker evidence, no raw category labels, no manifest schema change, no public-report schema change, and no execution-seed changes.
- Add aggregate-only non-surface grader-evidence coverage by difficulty so public cost-performance case-set gates cannot pass when structured/exact/numeric/citation evidence is concentrated in one difficulty bucket. Status: implemented in the no-spend case-set gate with a mechanical floor of five non-surface grader-evidence scored cases per closed-enum difficulty bucket, aggregate-only blocker evidence, no raw labels, no manifest schema change, no public-report schema change, and no execution-seed changes.
- Add verified case-manifest binding for evaluation runs so reports can prove which frozen case set was evaluated without leaking public digests. Status: implemented as `eval --case-manifest`, with HMAC required for holdouts, pre-spend digest verification, private-only binding digests, and no-digest public disclosure.
- Add run-provenance binding for evaluation runs so private reports fingerprint the executed resolved config, effective model-price snapshot, and fixed OpenRouter request policy without publishing stable linkability handles. Status: implemented as private `runProvenance`, CLI-generated resolved-config and effective-price fingerprints, closed OpenRouter request-policy fields plus a private digest, no path/argv/env capture, no-digest public disclosure, and public-report claim-gate blockers for missing run provenance or missing OpenRouter request-policy provenance.
- Add normalized CLI invocation metadata only where it helps private auditors orient a report. Status: implemented as optional private `runProvenance.invocation` with self-reported closed enums for `cli eval`, explicit raw-argv/path/environment omission disclosures, no raw option values, no output-write claims, and no public projection.
- Add a no-spend whole-run eval preflight so users can see planned call attempts and conservative completion-token cost before accidental spend. Status: implemented as `eval --preflight`, with no API-key requirement, no evaluation report writes unless `--preflight-out` is supplied, aggregate-only privacy defaults, optional call/cost guard flags, configured run budget ceiling, completion-cost upper bounds from the loaded price snapshot, and prompt-cost estimates explicitly limited to known initial request prompts.
- Add a strict deterministic choice grader for closed-label classification and routing-decision cases without adding LLM judges or substring matching. Status: implemented as `grader.choice`, with normalized whole-answer matching only, `choice_valid` and `choice_expected` checks, pre-spend validation of bounded unique labels, public report schema v11 check-kind projection, and docs warning that closed-label choice is mechanical evidence rather than semantic classification quality.
- Make CLI help a safe no-spend path so public users can inspect commands without accidentally triggering API-key checks, model snapshot writes, case-file reads, or runtime construction. Status: implemented for top-level `--help`/`-h` and command-specific help for `models`, `ask`, `validate-cases`, and `eval`, while missing arguments and unknown commands still fail closed with usage errors.
- Reject managed OpenRouter router, latest-resolution, and model-variant aliases from ordinary fixed-baseline configs. Status: implemented by rejecting managed router slugs such as `openrouter/fusion`, `openrouter/auto`, `openrouter/free`, `openrouter/pareto-code`, and `openrouter/bodybuilder`, latest-resolution aliases beginning with `~`, and any model-variant suffix such as `:online`, `:nitro`, or `:exacto`, while leaving concrete model IDs for a separately validated price snapshot path.
- Disable hidden OpenRouter default plugins for fixed-baseline calls. Status: implemented in the OpenRouter client by sending request-level disabled plugin entries for web search, response healing, context compression, Fusion, and Pareto Router, opting into router metadata, and failing closed unless successful responses report `strategy: "direct"` and no pipeline stages.
- Require provider endpoint pinning for public cost-performance report gates. Status: implemented with validated optional `provider.order`, OpenRouter request serialization, private run-provenance v2 binding of provider routing evidence into the resolved-config digest, private-only ordered slug/fallback/count evidence, no-digest public disclosure of only the closed endpoint-pinning status, and `provider_endpoint_pinning_missing` blockers unless the run reports one full endpoint slug with `allow_fallbacks: false`.
- Add a deterministic JSON schema-subset grader for stricter structured-output tasks without adding LLM judges or claiming full JSON Schema compliance. Status: implemented as `grader.json.schemaSubset`, with pre-spend schema validation, bounded recursive schema shape, symbolic mismatch diagnostics, public `json_schema_subset` check kinds, and docs warning that schema-shape checks are mechanical evidence only.
- Align the aggregation ledger with Fusion-like comparison mechanics without adding adaptive routing. Status: implemented as prompt/schema contract `frugal-fusion-prompts-v2`, with required `coverageGaps` and compact `uniqueAdoptedClaimIds`, prompt guidance that uniqueness is not evidence, and post-aggregation validation that ledger claim IDs refer only to blinded candidate claims.

## Design Review Disposition

SubAgents and, where earlier session policy required it, local role-based review passes covered architecture, evaluation, and security/cost. Material findings and dispositions:

- Budget exhaustion could become degraded success. Fixed by making budget errors terminal and adding a regression test.
- Raw tasks were retained in traces. Fixed by constructing trace request objects explicitly and adding a regression test.
- Missing self-review and repeated-sampling baselines. Fixed in the evaluation runner.
- Candidate order and identity leaked to the aggregator. Fixed with blinded aliases and deterministic counterbalancing.
- OpenRouter provider privacy defaults could be dropped by partial override. Fixed by merging provider defaults.
- Missing/zero usage and suspicious costs could fail open. Fixed by requiring token usage and positive prices, and using the greater of provider-reported and locally calculated cost.
- Grading was weak. Partially fixed by making deterministic graders explicit and failing ungraded non-smoke cases; richer domain graders remain future work.
- Trace privacy still retained too much prompt/answer content. Fixed by defaulting evaluation reports to prompt hash/length and result metadata only; raw/redacted prompt and output retention require explicit opt-in.
- Tiny token budgets could over-allocate across multi-call modes. Fixed by failing when the total completion-token cap cannot fund every planned call.
- `repeated` did not enforce `maxCandidates`. Fixed with shared candidate-budget validation.
- Model-client usage metadata was trusted too much. Fixed with orchestrator-level validation against planned model, finite token/cost values, status, and local price calculation.
- Smoke-only cases counted as task passes. Fixed by excluding smoke from task pass, cost/pass, and fusion harm metrics while adding smoke completion rate.
- Repeated sampling was role-prompted rather than true same-prompt sampling. Fixed by using the same repeated-sample candidate prompt for both calls and recording sampling metadata.
- Config parsing could allow malformed booleans/sampling/provider fields. Fixed with runtime config validation.
- Candidate-provided IDs could leak through metadata. Fixed by retaining only hashes of model-supplied candidate IDs.
- Regex graders were validated after spend. Fixed by validating graders before any model calls.
- Invalid-output provider responses could undercount spend. Fixed by extracting usage before content/schema validation and attaching usage to structured-output errors.
- Provider generation IDs could act as lookup handles. Fixed by dropping provider IDs from retained eval reports by default; retention is opt-in.
- Constraints were omitted from case-set hashes. Fixed by hashing constraint metadata into case-set identity and trace request metadata.
- Malformed nested config sections could silently fall back to defaults. Fixed by rejecting non-object nested sections.
- `self_review` ignored `maxRepairRounds=0`. Fixed by requiring at least one repair round for that mode.
- Model display-name leaks could reach the aggregator. Fixed by scrubbing snapshot model names and aliases case-insensitively.
- Repeated evaluation attempts lacked first-class trial identity. Fixed with `trialsPerCase`, trial-aware seed-material hashes, trial-aware order rotation, and execution-position counts.
- Duplicate evaluation config IDs corrupted metrics. Fixed by rejecting duplicate configs and using `trialsPerCase` for repetition.
- Aggregate pass-rate comparisons were unpaired. Fixed with pair-level win/loss/tie, harm, pass-rate delta, cost delta, and incremental cost-per-additional-pass metrics versus `direct`.
- Confidence intervals could imply independent attempts. Fixed by case-cluster bootstrap intervals that keep all trials for a sampled case together and warn on tiny scored case sets.
- Public report claim gates could pass interval cells whose confidence-interval method, level, or resample metadata was malformed or underpowered. Fixed by requiring case-cluster bootstrap, 95% level, and at least 500 resamples with boolean/count-only blocker evidence.
- Malformed confidence-interval metadata could still leak through public metrics even when claim-gate evidence was sanitized. Fixed by projecting only the known confidence-interval method and level, non-negative integer resample counts, and known generated bootstrap warnings; malformed values become `null` or are dropped before public serialization.
- Paid partial panel failures could be undercounted. Fixed by carrying paid usage from failed candidate calls into degraded results and zero-usable-candidate errors.
- Failed-candidate usage could bypass usage validation. Fixed by validating failed-call usage against the planned model, local price floor, token cap, finite values, and budget checks before it can affect cost accounting.
- Failed evaluation messages could leak raw provider or prompt-adjacent text through case outcomes. Fixed by storing status-derived messages by default and requiring explicit `retainFailureDetails` for bounded redacted details.
- Paired metrics and confidence intervals could show `0` for unavailable comparisons. Fixed by returning `null` for unavailable paired metrics and CI values when there is no scored population.
- Deterministic graders were too text-centric for structured and numeric tasks. Partially fixed with strict JSON path checks, finite numeric checks, global graded-answer size caps, a narrow regex subset, and symbolic grader diagnostics.
- `retainOutputs=true` could retain unbounded raw answers. Fixed by truncating retained answer and ledger strings and storing original length plus SHA-256 metadata.
- Public sharing of internal reports could leak model/provider IDs, case IDs, hashes, traces, answer keys, and exact small-sample costs. Fixed by adding an allowlisted public report projection, generated case IDs, disclosure notes, private-hash omission, small-sample cost/latency suppression, and CLI `--public-out` behavior that avoids silently writing a private default report.
- A larger example case set could be mistaken for a benchmark. Mitigated by naming it a public sample evaluation set, adding no-spend validation, and documenting that public claims still require a frozen holdout split.
- No-spend validation could leak private taxonomy through raw category labels. Fixed by making CLI validation output omit category labels by default while retaining a `--private` local-review option.
- Smoke-only cases could bypass malformed grader validation. Fixed by validating any supplied grader object shape even when deterministic grading is not required.
- Public row-level reports could be mistaken for unlinkable case anonymity. Mitigated by omitting raw `caseIndex`, labeling case identity as generated row labels, and documenting that public ordered case files make row-level outcomes linkable.
- All-smoke evaluations could report `task_pass_rate: 0` despite having no scored denominator. Fixed by reporting task pass rate as `null` when a config has zero scored attempts.
- Case-set fingerprints could create a second, inconsistent identity system. Fixed by using one canonical sorted-JSON fingerprint implementation for manifests, private evaluation `caseSetHash`, and manifest binding; HMAC-bound holdouts omit the unkeyed `caseSetHash`.
- A manifest fingerprint could conflict with public-report hash omission. Mitigated by keeping manifests as separate explicit artifacts and documenting that they should be published only for intentionally public or frozen case sets.
- Manifest validation could be mistaken for claim approval. Mitigated by using `claimReadiness.status` values that remain conservative and warnings that say no-spend validation cannot prove no tuning leakage or difficulty balance.
- Case-set identity changes could perturb execution seeds. Fixed by keeping report/manifest identity separate from model-visible execution seed material, which now depends on case id, task hash, and constraints only.
- Manifest flags and paths could leak more than their names suggest. Fixed by splitting `--public-category-labels` from `--public-case-ids`, omitting source paths by default, and requiring `--source-label` for checked-in public labels.
- Standalone public reports could look claim-ready and cost point estimates could appear without uncertainty. Fixed by adding public-report `claimReadiness` warnings and exposing rounded cost-per-pass intervals alongside public cost point estimates.
- Public report readers could still miss that direct/fusion comparison evidence is incomplete. Fixed by adding a machine-readable public-report `claimGate` that blocks incomplete direct-vs-fusion pairing, suppressed/missing intervals, suppressed cost evidence, and categories below the claim floor while keeping directional comparison descriptive.
- Public report claim gates could pass with only direct/fusion evidence even though the MVP comparison matrix includes self-review and repeated baselines. Fixed by requiring `direct`, `self_review`, `repeated`, and `fusion` in public-report claim gates, plus scored attempts, task pass-rate intervals, and cost-per-pass intervals for each required configuration while keeping directional comparison based on direct-vs-fusion deltas.
- Public report claim gates could pass when a required configuration had only a token scored-attempt cell rather than coverage over the same scored case-trials. Fixed by requiring each required configuration's scored-attempt count to equal the public report's scored trial count, and by returning task pass-rate intervals only when a configuration covers every scored case-trial in the interval population.
- Runtime callers could pass explicit `undefined` fields that validate but fail canonicalization. Fixed by rejecting undefined values during case validation.
- Public category metrics could imply named-domain performance or leak taxonomy. Mitigated by using generated-order labels, labeling them pseudonymous rather than anonymous, suppressing the whole breakdown if any category has fewer than five scored cases, omitting category cost/CI, and adding grader-mix counts plus claim-readiness caveats.
- Grader profile counts derived from runtime checks would disappear on failed outcomes and misrepresent the planned case set. Fixed by computing `graderEvidenceTier` from case definitions before model calls and storing only the closed enum on each case result.
- A "strength" label could overclaim benchmark quality. Fixed by using the neutral `graderEvidenceTier` name, documenting that tiers describe mechanical evidence shape rather than semantic grading strength, and adding a separate classifier version.
- Category-level pass rates could still hide grader mix confounding. Mitigated by publishing category-level tier counts, dominant tier/share, mixed-tier status, and small profile-cell warnings when category breakdowns are available.
- Smoke-only cases with configured graders could inflate apparent grader coverage. Mitigated by classifying every smoke-only case as `smoke_only` while reporting counts of configured grader kinds that runtime smoke grading ignores.
- Grader tier metadata could become a privacy side channel. Mitigated by deriving tiers only from closed structural families, never from raw grader strings, JSON paths, regex literals, expected values, prompts, or answers.
- Top-level public reports could lack grader evidence warnings when category breakdowns are suppressed. Fixed by adding scored tier counts, dominant scored tier, profile-mix status, and small profile-cell warnings to top-level `grader_evidence`.
- Public claim gates could pass with too little non-surface scored grader evidence if sample size, manifest, pairing, intervals, and provenance were otherwise present. Fixed by adding `insufficient_non_surface_grader_evidence` blockers to both case-set and public-report claim gates, using only aggregate tier counts.
- Non-surface grader evidence could be concentrated in one category while other claim-eligible categories stayed surface-text-only. Fixed by adding `category_non_surface_grader_evidence_underpowered` blockers derived from aggregate per-category coverage counts; no-spend public output still omits raw category labels and category-by-tier tables.
- Exact duplicate scored cases could be relabeled with different IDs or categories and still inflate denominators, confidence intervals, and category evidence. Fixed in the no-spend case-set claim gate by counting duplicate scored content over task, constraints, smoke flag, and grader while excluding ID/category from the internal fingerprint; only aggregate counts are exposed, and smoke-only duplicates do not block.
- Near-duplicate scored cases could evade the exact duplicate fingerprint while still inflating denominators and confidence intervals. Partially fixed in the no-spend case-set claim gate with a conservative lexical token-set similarity blocker over scored task and constraint text only, using aggregate count-only evidence and excluding smoke-only cases.
- Difficulty relabeling could change public claim-gate results without changing manifest identity. Fixed by adding difficulty to canonical case and manifest identity, bumping the case-set manifest schema to `frugal-fusion-case-set-manifest-v4` and the manifest fingerprint to `case-set-canonical-v3`, while keeping difficulty out of duplicate-content fingerprints and execution seed material.
- Public cost-performance case-set gates could pass with no difficulty metadata or with all cases in one difficulty bucket. Fixed by adding aggregate difficulty coverage counts and blockers for missing scored-case difficulty and fewer than 30 scored cases in any closed-enum difficulty bucket. Documented as a coverage floor rather than semantic difficulty calibration.
- Aggregate difficulty floors could still pass when hard, medium, or easy cases were concentrated in different categories. Fixed by adding a no-spend category-by-difficulty coverage blocker over claim-eligible categories only, exposing aggregate counts rather than raw taxonomy labels. Documented as a minimum coverage floor, not proof of semantic balance or difficulty calibration.
- Invalid runtime modes could fall through to the fusion path. Fixed by validating modes in both CLI `ask --mode` parsing and the orchestrator boundary before model selection.
- Future private metric records could leak extra keys through public projection helpers. Fixed by validating public configs and projecting metric records by `report.configs` rather than copying arbitrary record entries.
- Subset evaluations had metric record types that implied unavailable mode keys existed. Fixed by materializing all mode keys in private aggregate metrics while public reports still expose only executed configs.
- Default manifest row hashes include unsalted grader values. Mitigated by making that risk explicit in manifest privacy metadata and adding an HMAC private-audit manifest mode for private holdouts.
- HMAC manifest values could be confused with ordinary SHA-256 hashes or public proof. Fixed by using distinct `*HmacSha256` fields, `fingerprint.algorithm: "hmac-sha256"`, HMAC-specific privacy metadata, and claim-readiness warnings that verification requires the secret.
- HMAC manifests could still leak labels through explicit disclosure flags or direct builder calls. Fixed by rejecting `sourcePath`, case IDs, category labels, raw SHA digests, and `--private` summaries in HMAC mode at both CLI and exported builder boundaries.
- Weak or logged HMAC keys would collapse private-audit privacy. Fixed by reading the key only from an environment variable, requiring at least 32 bytes, not storing the env var name or key, and using domain-separated HMAC purposes for raw file, case set, rows, and raw lines.
- Large exact or JSON equality grader strings could reach fingerprinting before bounded validation. Fixed by applying the same bounded string limit used for other grader terms.
- A claim-readiness gate could be mistaken for benchmark approval. Fixed by naming the passing state `case_set_constraints_met`, keeping `overallClaimStatus: "external_evidence_required"`, and documenting that the gate can block claims but cannot prove model performance, pricing, or tuning isolation.
- Claim-gate CI output could leak private taxonomy if combined with `--private` or disclosure flags. Fixed by rejecting `--private --claim-gate`, source labels, public case IDs, and public category labels before file reads in claim-gate mode, and by keeping claim-gate finding evidence to aggregate counts rather than raw category names, case IDs, prompts, or grader values.
- A same-command manifest could be mistaken for public provenance. Mitigated by treating manifest presence as a case-set blocker only and separately requiring an archived manifest bound to the report in `externalEvidenceRequired`.
- A blocked claim gate could still write a disclosure-heavy manifest before exiting. Fixed by assessing the gate before manifest writes and skipping manifest output when blockers are present.
- A public-report gate could leak raw taxonomy or private model keys if it inspected the private report directly. Fixed by deriving the report gate only from `PublicEvalMetrics`, public configs, and no-digest public disclosure metadata after allowlist projection.
- Public-report claim gates could pass malformed manually constructed public disclosures if redaction and omission literals were trusted only through TypeScript types. Fixed by adding a claim-gate disclosure-contract blocker that checks top-level redacted/omitted fields, strict `not_provided` shapes, and nested manifest/run-provenance digest/path omission flags with boolean-only evidence; adjacent malformed enum/status evidence is sanitized to known public values or `null`.
- Adding `claimGate` and later category interval/coverage diagnostics, public choice check kinds, request-policy disclosure, and provider endpoint pinning disclosure to public reports changes the top-level public artifact contract. Addressed by bumping the public report schema literal, now `frugal-fusion-public-eval-v11`.
- `auto` accepted by the API could be mistaken for a Fugu-style router. Fixed by keeping it direct-only, recording closed-enum `autoRouting` metadata only for `mode: "auto"`, and documenting that adaptive routing is deferred until fixed-panel traces justify it.
- Citation checks could be mistaken for source grounding or entailment. Fixed by naming them citation-mechanics checks, requiring at least one positive citation requirement, keeping source corpus text out of the grader config, bounding literal snippets, using symbolic diagnostics only, and documenting that prompts may still contain source snippets governed by prompt-retention/public-report rules.
- Citation required-claim checks could pass with substantial intervening prose between the literal claim and the required source citation. Mitigated by adding an optional per-claim immediate placement mode that requires the source citation to be the first non-whitespace text after the claim snippet while keeping the default bounded-window behavior for existing cases.
- Claim-gate validation errors could leak private row identifiers to CI stderr before public aggregate blockers are emitted. Fixed by returning generic public JSON for claim-gate parse/validation failures and by making private holdout claim gates require HMAC manifests rather than unsalted SHA-256 manifests.
- Public reports could claim manifest binding without a private report or leak unkeyed holdout hashes. Fixed by persisting a private report whenever `eval --case-manifest` is used with `--public-out`, storing HMAC holdout binding digests only in the private report, setting `caseSetHash` to `null` for HMAC-bound reports, and projecting only no-digest binding status to public reports.
- Public reports could pass evidence-shape checks while the exact private holdout's no-spend case-set gate was blocked. Fixed by deriving the private `caseSetClaimGate` attestation inside `runEvaluation` from the verified manifest binding and current cases, rejecting caller-supplied attestations, projecting only no-digest pass/blocked status publicly, requiring that status for `public_report_constraints_met`, and failing manifest-bound public-report CLI runs before model spend when the no-spend gate is blocked.
- Public config/model/request-policy digests would create stable linkability handles and could invite guessing against known local files or private account settings. Fixed by keeping run-provenance digests private, projecting only no-digest status, omitting paths/argv/env/source kinds publicly, and keeping the public report schema on the current `frugal-fusion-public-eval-v11` contract.
- Raw config files can contain dormant secrets or private labels in unknown keys. Fixed by fingerprinting only allowlisted resolved config fields after defaults are applied, and by treating exact raw files as separate private audit artifacts rather than embedding or hashing them by default.
- Full model catalog fingerprints over-capture provider metadata and create unnecessary churn. Fixed by fingerprinting the sorted unique effective `PriceSnapshotEntry` records planned for the run instead of the whole `--models` file.
- Duplicate model IDs in loaded price snapshots could silently overwrite earlier prices and make cost accounting ambiguous. Fixed by rejecting duplicate `PriceSnapshotEntry.modelId` values at registry ingestion while still allowing repeated role-specific requests for the same validated model entry.
- Fusion configs could use the same candidate model twice and blur the line between heterogeneous fusion and the same-model `repeated` baseline. Fixed by requiring the two `models.candidateModels` entries to be distinct while still allowing one candidate, the direct model, and the aggregator to share a cheap model when that is the intended panel design.
- Managed OpenRouter router, latest-resolution, and model-variant aliases could silently turn fixed-baseline cells into external managed-routing, retrieval, version-resolution, or provider-sorting comparisons. Fixed by rejecting known managed router slugs, `~` latest aliases, and colon model variants in fixed-baseline model roles; future managed Fusion/search/router references should use separately labeled external-reference runners.
- Account-level OpenRouter plugin defaults could add search, request mutation, response healing, context compression, Fusion, or Pareto routing behind ordinary model calls. Fixed by sending explicit disabled plugin entries, requiring router metadata to confirm direct routing with no pipeline stages, and binding that request policy into private run provenance with only no-digest public disclosure. Organization settings that prevent overrides can still make such runs fail closed or require a separately labeled external condition.
- OpenRouter's default and account-level provider routing could load-balance or fall back across provider endpoints while the report looked like a fixed baseline. Fixed public-claim readiness by binding `provider.order` into private run-provenance v2 and requiring the public no-digest status `single_provider_endpoint_pinned`, which is derived only from one full endpoint slug and `allow_fallbacks: false`. Private auditors still need the exact config, report digest, account/BYOK settings, and provider routing evidence; base provider slugs and multiple ordered providers remain exploratory but block public claims.
- Normalized invocation metadata could be mistaken for command proof or leak raw flags through "helpful" option capture. Fixed by making it self-reported private metadata, accepting only a closed `cli eval` shape, rejecting extra raw argv/env/path keys before spend, omitting output persistence claims, and keeping it out of public reports and claim gates.
- Adding optional private invocation metadata could force unnecessary public schema churn. Treated run-provenance v1 as an extensible private schema for optional metadata that does not change the required resolved-config or model-price fingerprint contract, and documented that public reports still expose only no-digest required-field status.
- Aggregator prompt/schema changes could be mislabeled as the same evaluation condition. Fixed by deriving runtime metadata from the active prompt contract, rejecting stale configured prompt versions, bumping the default and example prompt contract to `frugal-fusion-prompts-v2`, and adding regression assertions that stale prompt metadata is rejected and new runs record v2.
- Aggregator ledgers could cite stale, candidate-supplied, invented, duplicate, or contradictory claim IDs. Fixed by validating `consensusClaimIds`, `adoptedClaimIds`, `uniqueAdoptedClaimIds`, `rejectedClaims`, and conflict claim IDs against the blinded candidate claim set, rejecting duplicate ledger IDs, rejecting adopted/rejected overlap, and requiring consensus claim IDs to span at least two candidates; semantic ledger failures are treated as paid invalid aggregator output.
- Aggregator schema-invalid responses could lose stage/model failure metadata. Fixed by normalizing aggregator failures for schema and semantic invalid-output paths while preserving paid usage and, when available, provider response IDs in the call trace.
- Unique-claim tracking could reward novelty or prompt injection. Mitigated by storing only compact adopted claim IDs, requiring unique adopted IDs to be adopted and non-consensus when reported, and prompting that uniqueness is not evidence; unsupported unique claims still belong in rejected claims or required checks. The runtime validates ledger ID consistency, while semantic completeness of coverage and unique-insight judgments remains model-reported metadata rather than public claim evidence.

## Remaining Public-Readiness Risks

- The public sample case set is larger and equal-count across coarse categories, with author-supplied difficulty coverage metadata balanced two `easy`, two `medium`, and two `hard` cases per category plus synthetic citation-mechanics and strict choice coverage, but it remains small, surface-text heavy, and not a frozen benchmark. Public cost-performance claims need a holdout split, versioned case manifest, calibrated difficulty review, and stronger grader coverage.
- The deterministic graders are intentionally small. Choice, JSON path/schema-subset, numeric, and citation-mechanics graders cover common closed-label, structured, and source-citation mechanics tasks, including optional immediate citation placement, but richer code and semantic source-entailment graders are still needed before broader claims.
- Grader evidence tiers reveal that the current public sample still leans surface-text heavy even after adding citation-mechanics coverage. They are useful disclosure metadata, and claim gates now require at least a small non-surface scored evidence cell, but they are not a substitute for richer graders or a balanced holdout.
- Category-level non-surface evidence floors are still mechanical evidence-shape checks. They reduce obvious public-claim weakness but do not prove category difficulty balance, semantic grading quality, or source entailment.
- Difficulty coverage floors are metadata coverage checks only. They do not prove the labels are calibrated, that a holdout is semantically balanced, or that task difficulty is comparable across categories.
- Category-by-difficulty coverage floors reduce obvious difficulty/category confounding in claim-eligible categories, but they are still mechanical metadata checks. They do not prove semantic balance, label calibration, or equal task complexity within each cell.
- Non-surface grader-evidence floors by difficulty reduce obvious evidence-shape concentration across easy/medium/hard buckets, but they remain mechanical coverage checks. They do not prove semantic difficulty calibration, grader quality, source entailment, or equal task complexity across buckets.
- Duplicate scored-content blocking catches literal duplicates of task, constraints, smoke flag, and grader plus high-confidence lexical near duplicates over task and constraints. It does not catch paraphrases, semantic near-duplicates, shared answer keys, topical clustering, or holdout contamination from tuning.
- The checked-in public sample manifest freezes identity but does not prove benchmark validity. Holdout process evidence, stronger graders, and category-level results are still required for public performance claims.
- Sampling seeds are best-effort. Provider/model changes can still change outputs.
- The preflight token estimate is conservative enough for local gating work but not a tokenizer-accurate cost guarantee; actual post-call usage remains authoritative.
- The whole-run `eval --preflight` completion-cost value is an upper bound only for configured max completion tokens under the loaded price snapshot. Prompt costs for generated-output-dependent stages are not knowable before spend, and pricing/model availability can still change after the snapshot is fetched.
- The current model snapshot and preflight logic are token-price only. They do not account for managed Fusion inner tool loops, web-search/fetch usage, retrieval charges, or hidden deliberation usage, so managed Fusion/search references need a separate runner and expanded price/usage schema before public comparison.
- Provider defaults reduce accidental routing drift, but they do not fully pin exact provider endpoints. Public cost-performance report gates now require private run provenance to report exact endpoint pinning through one full `provider.order` slug and `allow_fallbacks: false`; exploratory runs without that evidence remain valid but blocked for public claims.
- Router metadata is requested for public-readiness checks, but exact endpoint custody still depends on the private report and external reproduction package. Cache hits or organization-enforced plugins that prevent metadata-based confirmation should be treated as fixed-baseline blockers rather than silently accepted evidence.
- Default private evaluation reports still include model/provider identities in usage and price snapshots. This is useful for reproducible benchmark claims, but teams evaluating private endpoints should share the public report projection unless they intend to disclose provenance.
- Default reports retain failure status categories but not raw failure messages. Detailed failure messages are for local debugging only.
- Confidence intervals are descriptive for small samples. Public claims should still use a larger balanced case set and report category-level results.
- Public reports still omit raw category labels to avoid taxonomy leakage. When category-level public reporting meets the disclosure floor, public reports may include generated-order category labels that are pseudonymous and potentially linkable, not anonymous.
- HMAC private-audit manifests reduce public content linkability, but they still expose structural metadata and require careful shared-secret custody. They are not a replacement for a documented holdout process.
- The claim gate checks case-set constraints only. Passing it still requires external holdout-process evidence, a frozen and archived manifest bound to the report, confidence intervals, and private reproduction artifacts before any public cost-performance claim.
- The public-report claim gate checks public artifact evidence plus the private report's no-digest manifest-binding, no-spend case-set gate, and run-provenance assertions. Passing it still does not independently prove holdout isolation, model/provider provenance, price provenance, or auditor access to the private manifest/HMAC materials.
- The public-report claim gate now blocks missing no-digest run-provenance disclosure, but that disclosure still asserts only that the private report carries comparable fingerprints. Passing the gate does not independently prove config custody, model/provider provenance, price provenance, command reproducibility, or auditor access to the private artifacts.
- Public reports could carry a no-digest run-provenance disclosure whose evaluated-config count disagreed with the public matrix. Fixed by adding a public-report claim-gate consistency blocker for count mismatches against both the public report configs and the fixed MVP matrix, while still omitting config identities and digests publicly.
- Manually constructed public reports could rely on indirect blockers for unknown, duplicate, or extra public config IDs. Fixed by adding an explicit public-config contract blocker with boolean/count-only evidence before the claim gate builds its internal config set, so malformed config lists block without echoing raw config strings.
- Private normalized CLI invocation metadata is self-reported report context, not a signed command log. Exact command lines, external run logs, and file custody still belong in the private reproduction package outside the public report.
- Current public category breakdowns are useful for robustness auditing, and now include grader evidence tier counts plus within-category task pass-rate and pass-rate-delta intervals, but they still rely on author-supplied difficulty metadata and limited deterministic grader coverage; richer graders, calibrated difficulty review, and category-level cost intervals remain future work.
- `auto` is not yet a learned or heuristic router. Future routing should be evaluated offline against fixed direct/self-review/repeated/fusion traces before becoming a public mode.
- `frugal-fusion-prompts-v2` is a new prompt/schema condition. Do not combine v1 and v2 runs in the same headline comparison without labeling the prompt contract, and do not treat the new comparison ledger as semantic source entailment or public claim approval.

## Role Review Fallback

Early setup happened under a local role-review fallback because SubAgents were not available in that moment; subsequent design and implementation reviews used SubAgents once the workflow exposed them.

### Architecture Review

- Keep modules narrow: provider interface, registry, orchestration, evaluation, CLI.
- Use typed failures for timeout, provider error, invalid output, and budget exhaustion.
- Avoid dynamic routing and third candidates in this slice.

### Evaluation Review

- Persist price snapshots with each case result.
- Include direct and fusion on identical cases.
- Report harm rate and partial failures immediately, even before adding richer graders.

### Security And Cost Review

- Redact secrets before storage.
- Do not persist raw prompts by default.
- Treat candidates as untrusted JSON data in aggregator prompts.
- Fail closed on missing or stale price data.

## Success Criteria

- `pnpm format`, `pnpm lint`, `pnpm typecheck`, and `pnpm test` pass.
- Fake-provider tests cover success, timeout, invalid output, provider error, partial panel failure, and budget exhaustion.
- CLI can run a single prompt or JSONL evaluation.
